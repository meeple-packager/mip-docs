# Добро пожаловать в MIP! 🚀

**MIP** (MInimal Package Manager) - это лёгкий и быстрый менеджер пакетов для Node.js, написанный на чистом JavaScript.

::: callout info "Что такое MIP?" icon:info
MIP - это альтернатива npm, pnpm и yarn. Он быстрее, проще и требует меньше ресурсов. В отличие от других менеджеров, MIP использует **глобальный кэш**, не создаёт `node_modules/` и поддерживает **плагины**.
:::

## 🎯 Ключевые возможности

- **🚀 Скорость** - установка пакетов в несколько раз быстрее благодаря параллельной загрузке, HTTP/2 и трёхуровневому кэшированию.
- **📦 Глобальный кэш** - пакеты хранятся один раз в `~/.mip/store/` и используются всеми проектами. Экономит место и ускоряет установку.
- **🔌 Плагины** - расширяйте функциональность MIP с помощью плагинов. Создавайте свои команды, подписывайтесь на события и интегрируйтесь с внешними сервисами.
- **📝 YAML-конфиги** - вместо `mip.json` теперь используется `mip.yml` и `mip-lock.yml`. Читаемые, с поддержкой комментариев.
- **🔒 Безопасность** - встроенный аудит зависимостей и проверка уязвимостей.
- **🌍 Мультиязычность** - поддержка 10 языков интерфейса.
- **📁 Монорепозитории** - встроенная поддержка workspaces.
- **🔧 Гибкость** - работа с глобальными и локальными пакетами, поддержка `package.json` для совместимости с npm.
- **🧩 Свой реестр** - возможность публиковать и устанавливать пакеты из собственного реестра.

## ⚡ Быстрый старт

```bash
# Установка MIP
git clone https://github.com/kiwinatra/mip
cd mip
npm install
npm run build

# Создание нового проекта
mip init
mip install express
mip run start
```

::: tip "💡 Совет"
После сборки MIP предложит добавить себя в `PATH`, чтобы вы могли использовать его из любой папки.
:::

## 🧩 Как это работает

MIP использует **глобальный кэш** (`~/.mip/store/`), где хранит все версии пакетов. Это позволяет:
- Устанавливать пакеты без дублирования.
- Мгновенно переключаться между версиями.
- Работать без интернета (если пакет уже есть в кэше).
- Экономить до 90% дискового пространства.

**Отсутствие `node_modules/`** - вместо этого MIP использует симлинки на глобальный кэш и манифест `.mip/manifest.json` для отслеживания установленных пакетов.

Подробнее читай в разделе [Архитектура](/ru/dev/architecture).

## 📚 Команды

MIP включает все необходимые команды для работы с пакетами:

| Команда | Описание |
|---------|----------|
| `mip init` | Инициализация нового проекта |
| `mip install` | Установка зависимостей |
| `mip update` | Обновление пакетов |
| `mip uninstall` | Удаление пакетов |
| `mip list` | Просмотр установленных пакетов |
| `mip outdated` | Проверка устаревших пакетов |
| `mip audit` | Проверка безопасности |
| `mip run` | Запуск скриптов |
| `mip exec` | Выполнение бинарников |
| `mip global` | Управление глобальными пакетами |
| `mip workspaces` | Работа с монорепозиториями |
| `mip plugin` | Управление плагинами |
| `mip pe` | Выполнение команд плагинов |

Полный список команд и их описание - в разделе [Команды](/ru/commands/install).

## 📁 Структура проекта

```
my-project/
├── mip.yml             # Конфигурация проекта (YAML)
├── mip-lock.yml        # Зафиксированные версии пакетов (YAML)
├── .mip/
│   ├── manifest.json   # Манифест установленных пакетов
│   ├── cache/          # Кэш загрузок
│   └── packages/       # Локальные пакеты (если есть)
└── node_modules/       # Симлинки на глобальный кэш
```

## 🌍 Мультиязычность

MIP поддерживает 10 языков интерфейса. Чтобы сменить язык:

```bash
mip language ru   # Русский
mip language en   # Английский
mip language de   # Немецкий
mip language zh   # Китайский
```

## 🔌 Плагины

MIP имеет встроенную систему плагинов. Плагины позволяют расширять функциональность без изменения основного кода.

```bash
# Создать плагин
mip plugin create my-plugin

# Скомпилировать и активировать
mip plugin compile my-plugin
mip plugin activate my-plugin

# Выполнить команду плагина
mip pe my-plugin my-command
```

Подробнее: [Плагины](/ru/commands/plugin).

## ❓ Часто задаваемые вопросы

**Чем MIP отличается от npm?**
MIP быстрее, проще и не требует сложной настройки. Он использует глобальный кэш, не создаёт `node_modules/`, поддерживает плагины и YAML-конфиги.

**Можно ли использовать MIP в CI/CD?**
Да, для этого есть команда `mip ci`, которая оптимизирована для автоматических сборок и поддерживает `--frozen-lockfile`.

**Как обновить MIP?**
Достаточно пересобрать проект из репозитория и переустановить.

**Поддерживает ли MIP `package.json`?**
Да, MIP полностью совместим с `package.json`. Вы можете использовать MIP в любом проекте с `package.json`.

**Как мигрировать со старого `mip.json`?**
При первом запуске MIP автоматически мигрирует `mip.json` в `mip.yml` и создаёт бэкап.
