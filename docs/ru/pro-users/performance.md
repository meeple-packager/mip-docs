# Производительность MIP ⚡

MIP спроектирован для максимальной скорости. Вот как он этого достигает.

---

## 🚀 Ключевые технологии

### 1. Глобальный кэш

Все пакеты хранятся **один раз** в `~/.mip/store/` и используются всеми проектами.

```
~/.mip/store/
├── lodash/
│   └── 4.17.21/     # ← одна копия для всех проектов
└── express/
    └── 4.18.2/      # ← одна копия для всех проектов
```

**Преимущества:**
- Экономия места: 100 МБ вместо 1 ГБ (10 проектов)
- Ускорение установки: не нужно качать повторно

### 2. Параллельная загрузка

MIP скачивает до 10 пакетов одновременно, используя все ядра процессора.

```javascript
// lib/core/parallel-download.js
const maxWorkers = os.cpus().length;
const chunks = this.chunkArray(packages, maxWorkers);
const results = await Promise.all(chunks.map(downloadChunk));
```

### 3. HTTP/2

Использует HTTP/2 для мультиплексирования запросов. Одно соединение → много запросов.

### 4. Трёхуровневый кэш (SuperCache)

| Уровень | Хранилище | Скорость |
|---------|-----------|----------|
| L1 | Память (LRU) | Наносекунды |
| L2 | Диск (`.mip/cache/`) | Миллисекунды |
| L3 | In-flight (дедуп) | Микросекунды |

### 5. Отсутствие дублирования

В отличие от npm, MIP хранит каждую версию пакета **один раз** в `~/.mip/store/`.

### 6. Потоковая распаковка

Распаковывает tarball без сохранения на диск (экономит I/O).



## 🔧 Настройка производительности

### Увеличение числа параллельных загрузок

По умолчанию — количество ядер CPU. Можно изменить в коде:

```javascript
// lib/core/parallel-download.js
this.maxWorkers = options.maxWorkers || os.cpus().length;
```

### Очистка кэша для тестов

```bash
mip cache clean
mip install express
```

### Мониторинг скорости

Включи debug:

```bash
DEBUG=1 mip install express
```

## 📈 Сравнение с другими менеджерами

### npm
- Дублирует пакеты в каждом проекте
- Нет параллельной загрузки (по умолчанию)
- Медленный на повторных установках

### pnpm
- Быстрый, использует глобальный стор
- Использует симлинки (иногда проблемы)
- Сложная структура

### MIP
- Простая структура — симлинки на `~/.mip/store/`
- Параллельная загрузка + HTTP/2
- Трёхуровневый кэш
- Работает офлайн

---

## 💡 Советы по оптимизации

1. **Используй `mip ci` в CI** — быстрее, чем `mip install`
2. **Кэшируй `~/.mip/store/` в CI** — ускоряет повторные сборки
3. **Очищай кэш** когда он становится слишком большим
4. **Используй `--ci`** для неинтерактивной установки
5. **Обновляй MIP** — каждая версия быстрее

## 🧠 Почему MIP такой быстрый?

1. **Глобальный кэш** — не дублирует пакеты между проектами
2. **Параллельность** — загружает пакеты одновременно
3. **Кэширование** — не делает лишних запросов
4. **HTTP/2** — меньше накладных расходов
5. **Потоковая распаковка** — без временных файлов
6. **Дедупликация** — удаляет лишние версии
