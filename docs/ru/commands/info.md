# mip info

::: callout info "Что это?" icon:info
`mip info` - это утилита для быстрого получения информации о любом пакете из реестра npm. Она показывает описание, автора, версии, ссылки на домашнюю страницу и репозиторий, а также дату публикации. Это удобно для изучения нового пакета перед установкой, проверки актуальности версии или поиска ссылки на документацию.
:::

## Синтаксис

```bash
mip info <package-name>
```

## Аргументы

| Имя | Тип | Обязательный | Описание |
|-----|-----|--------------|----------|
| `package-name` | строка | ✅ | Имя пакета, информацию о котором нужно получить |

## Логика работы

1. **Проверка аргументов** - если имя пакета не указано, выводится справка по использованию.

2. **Параллельный запрос** - одновременно выполняются два запроса к реестру:
   - Получение информации о последней версии пакета.
   - Получение списка всех доступных версий.

3. **Формирование отчёта** - из полученных данных собирается информационная карточка:
   - Имя пакета и последняя версия.
   - Описание.
   - Автор.
   - Домашняя страница.
   - Количество доступных версий.
   - Дата публикации.
   - Ссылка на репозиторий.

4. **Вывод последних версий** - показывает последние 5 версий с отметкой актуальной.

## Типичные ошибки

::: callout warning "❌ Package not found" icon:alert-triangle
**Причина:** Указанный пакет не существует в реестре npm.

**Решение:**
1. Проверьте правильность написания имени пакета.
2. Возможно, пакет приватный и требует авторизации.
3. Попробуйте поискать пакет: `mip search <partial-name>`
:::

::: callout warning "❌ Failed to fetch info" icon:alert-circle
**Причина:** Проблемы с сетью или реестр npm недоступен.

**Решение:**
1. Проверьте интернет-соединение.
2. Проверьте доступность реестра: `mip doctor`
3. Попробуйте позже.
:::

::: callout tip "💡 Совет" icon:lightbulb
`mip info` показывает информацию **из реестра**, а не из локально установленного пакета. Если вы хотите узнать версию локально установленного пакета, используйте `mip list` или посмотрите в `mip-lock.yml`.
:::
