# mip repo

::: callout info "Что это?" icon:github
`mip repo` — это интерактивный TUI-файловый менеджер для просмотра GitHub-репозиториев. В отличие от простой CLI-версии, TUI предоставляет две панели: слева — файловый менеджер, справа — просмотр содержимого файлов. Вы можете навигировать по папкам, читать README и скачивать файлы в удобном интерфейсе.
:::

## Синтаксис

```bash
mip repo <owner/repo> [--branch <branch>] [--downloadPath <path>]
```

## Аргументы

| Имя | Тип | Обязательный | Описание |
|-----|-----|--------------|----------|
| `owner/repo` | строка | ✅ | Владелец и имя репозитория (например, `kiwinatra/mip`) |

## Опции

| Флаг | Описание |
|------|----------|
| `--branch` | Ветка для просмотра (по умолчанию: `main`) |
| `--downloadPath` | Путь для сохранения скачанных файлов (по умолчанию: `download/`) |

## Управление TUI

### Навигация

| Клавиша | Действие |
|---------|----------|
| `Tab` | Переключить фокус между левой и правой панелью |
| `j` / `↓` | Вниз по списку (левая панель) или прокрутка вниз (правая панель) |
| `k` / `↑` | Вверх по списку (левая панель) или прокрутка вверх (правая панель) |
| `Enter` | Открыть папку или файл (в левой панели) |
| `q` / `Ctrl+C` | Выйти из TUI |

### Команды (вводятся внизу экрана)

| Команда | Описание |
|---------|----------|
| `help` | Показать справку |
| `ls` | Перезагрузить текущую папку |
| `cd <dir>` | Перейти в указанную папку |
| `cd ..` | Перейти на уровень выше |
| `view <file>` | Открыть файл в правой панели |
| `get <file>` | Скачать файл в папку `downloadPath` |
| `exit` или `q` | Выйти из TUI |

## Логика работы

1. **Подключение к GitHub API** — команда использует GitHub API для получения содержимого репозитория.

2. **Двухпанельный интерфейс**:
   - **Левая панель** — файловый менеджер (папки и файлы).
   - **Правая панель** — содержимое выбранного файла или README.

3. **Автоматическое открытие README** — при входе в папку справа автоматически отображается README (если он есть).

4. **Интерактивные команды** — вводятся в нижней строке, выполняются без перезагрузки интерфейса.

## Примеры

::: tabs

```bash
mip repo kiwinatra/mip
```

== tab "Навигация по папкам"
В TUI:
1. Используйте `j`/`k` для выбора папки `lib`.
2. Нажмите `Enter` — откроется папка `lib`.
3. Справа автоматически отобразится README из папки `lib` (если есть).
:::
== tab "Просмотр файла"
В TUI:
1. Выберите файл (например, `package.json`) с помощью `j`/`k`.
2. Нажмите `Enter` — содержимое файла отобразится в правой панели.
:::
== tab "Скачивание файла"
В TUI:
1. Переключитесь в режим ввода команд (нажмите `Tab`, если нужно).
2. Введите: `get package.json`
3. Файл сохранится в папку `download/`.
:::
== tab "Использование с токеном"
```bash
export GITHUB_TOKEN=ghp_xxxxx
mip repo kiwinatra/mip
```
:::

## Типичные ошибки

::: callout warning "❌ Repository not found" icon:alert-triangle
**Причина:** Указан неверный владелец или имя репозитория.

**Решение:**
1. Проверьте правильность написания: `owner/repo`.
2. Убедитесь, что репозиторий публичный (или у вас есть доступ).
:::

::: callout warning "❌ API rate limit exceeded" icon:alert-circle
**Причина:** Превышен лимит запросов к GitHub API (60 запросов/час для неавторизованных).

**Решение:**
1. Установите переменную `GITHUB_TOKEN` для увеличения лимита до 5000 запросов/час.
2. Подождите час, чтобы лимит сбросился.
:::

::: callout tip "💡 Совет" icon:lightbulb
`mip repo` в TUI-режиме идеально подходит для:
- **Быстрой навигации** по репозиторию без клонирования.
- **Чтения документации** в удобном интерфейсе.
- **Выборочного скачивания** файлов.

**TUI vs обычный режим:**
| Режим | Особенность |
|-------|-------------|
| TUI (по умолчанию) | Две панели, интерактивная навигация |

Для работы с приватными репозиториями установите `GITHUB_TOKEN` с правами на чтение.
:::
