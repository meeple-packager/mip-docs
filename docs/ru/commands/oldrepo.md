# mip repo

::: callout info "Что это?" icon:github
`mip repo` — это интерактивный файловый менеджер для GitHub-репозиториев. Вы можете просматривать содержимое репозитория, навигировать по папкам, читать README в полноэкранном режиме и скачивать файлы локально — всё это без необходимости клонировать репозиторий. Команда использует GitHub API и поддерживает аутентификацию через `GITHUB_TOKEN` для обхода ограничений.
:::

## Синтаксис

```bash
mip repo <owner/repo> [--branch <branch>] [--path <download-path>]
```

## Аргументы

| Имя | Тип | Обязательный | Описание |
|-----|-----|--------------|----------|
| `owner/repo` | строка | ✅ | Владелец и имя репозитория (например, `kiwinatra/mip`) |

## Опции

| Флаг | Описание |
|------|----------|
| `--branch` | Ветка для просмотра (по умолчанию: `main`) |
| `--path` | Путь для сохранения скачанных файлов (по умолчанию: `download/`) |

## Логика работы

1. **Подключение к GitHub API** — команда использует публичное API GitHub для получения содержимого репозитория.

2. **Аутентификация** — если установлена переменная окружения `GITHUB_TOKEN`, она используется для увеличения лимитов запросов.

3. **Интерактивный режим** — после входа в репозиторий открывается интерактивная оболочка со следующими командами:
   - `ls` — показать содержимое текущей папки.
   - `cd <folder>` — перейти в папку.
   - `cd ..` — перейти на уровень выше.
   - `readme` или `cat` — показать README в полноэкранном режиме.
   - `get <filename>` — скачать файл в локальную папку.
   - `help` или `?` — показать справку.
   - `exit` — выйти из просмотра.

mip repo kiwinatra/mip

Открывается полноэкранный режим для чтения README:
- `j` / `↓` — прокрутка вниз.
- `k` / `↑` — прокрутка вверх.
- `PgDn` / `PgUp` — страничная прокрутка.
- `q` — выход из режима чтения.

## Типичные ошибки

::: callout warning "❌ Repository not found" icon:alert-triangle
**Причина:** Указан неверный владелец или имя репозитория.

**Решение:**
1. Проверьте правильность написания: `owner/repo`.
2. Убедитесь, что репозиторий публичный (или у вас есть доступ).
:::

::: callout warning "❌ API rate limit exceeded" icon:alert-circle
**Причина:** Превышен лимит запросов к GitHub API (60 запросов/час для неавторизованных).

**Решение:**
1. Установите переменную `GITHUB_TOKEN` для увеличения лимита до 5000 запросов/час.
2. Подождите час, чтобы лимит сбросился.
:::

::: callout warning "❌ File not found" icon:alert-circle
**Причина:** Указанный файл не существует в текущей папке.

**Решение:**
1. Используйте `ls` для просмотра доступных файлов.
2. Убедитесь, что вы находитесь в правильной папке.
:::

::: callout tip "💡 Совет" icon:lightbulb
`mip repo` — это мощный инструмент для:
- **Быстрого просмотра** содержимого репозитория без клонирования.
- **Чтения документации** в репозиториях с большими README.
- **Выборочного скачивания** файлов (экономит время и трафик).

Для работы с приватными репозиториями установите `GITHUB_TOKEN` с правами на чтение.
:::

## Связанные команды

- [mip install](/commands/install) — установка пакетов (можно использовать с GitHub-репозиториями).
- [mip info](/commands/info) — информация о пакетах из реестра.

::: collapsible "🧠 Как работает интерактивный режим?"
После входа в репозиторий открывается **REPL-оболочка** (Read-Eval-Print Loop), где вы можете выполнять команды:

1. **`ls`** — запрашивает GitHub API для получения содержимого текущей папки.
2. **`cd <folder>`** — обновляет путь и повторно запрашивает содержимое.
3. **`readme`** — ищет файлы README в текущей папке (сначала ищет `README.md`, затем другие варианты) и открывает их в полноэкранном режиме.
4. **`get <file>`** — скачивает файл в папку, указанную в `--path`.

**Полноэкранный режим README:**
- Использует альтернативный экран терминала.
- Поддерживает навигацию с помощью клавиатуры.
- Автоматически подстраивается под размер окна.

**Автоматическое определение README:**
Команда ищет файлы в следующем порядке:
1. `README.md`, `README.MD`, `README.rst`, `README.txt`.
2. Любой файл, начинающийся с `readme.`.
3. Любой файл с расширением `.md`.
:::

::: collapsible "🔧 Использование GitHub API"
`mip repo` использует **GitHub REST API** для получения содержимого репозитория:

**Endpoint:**
```
GET /repos/{owner}/{repo}/contents/{path}
```

**Аутентификация:**
```bash
export GITHUB_TOKEN=your_personal_access_token
```

**Лимиты запросов:**
- **Неавторизованные:** 60 запросов/час.
- **Авторизованные:** 5000 запросов/час.

**Формат ответа:**
```json
[
  {
    "name": "README.md",
    "path": "README.md",
    "type": "file",
    "content": "base64...",
    "encoding": "base64"
  },
  {
    "name": "lib",
    "path": "lib",
    "type": "dir"
  }
]
```

**Важно:** Для файлов содержимое приходит в формате Base64 и декодируется на лету.
:::
