# mip dedupe

::: callout info "Что это?" icon:scissors
`mip dedupe` находит и устраняет дублирующиеся версии пакетов в вашем проекте. Это помогает уменьшить размер, ускорить установку и избежать конфликтов версий. Команда предлагает два режима работы: **быстрый** (по умолчанию) и **полный** - для максимальной оптимизации.
:::

## Синтаксис

```bash
mip dedupe [--full]
```

## Опции

| Флаг | Описание |
|------|----------|
| `--full` | Включает **полный режим** дедупликации - более агрессивно перестраивает дерево зависимостей, удаляя все возможные дубликаты. Может занять больше времени. |

## Логика работы

1. **Анализ lock-файла** - читает `mip-lock.yml` и строит граф зависимостей.

2. **Поиск дубликатов** - находит пакеты, у которых:
   - Одинаковое имя.
   - Разные версии.
   - Обе версии удовлетворяют требованиям всех родительских пакетов.

3. **Проверка совместимости** - определяет, можно ли заменить старую версию на новую без нарушения совместимости (проверяет major-версию).

4. **Вывод отчёта** - показывает список найденных дубликатов с указанием:
   - Имя пакета и его версии.
   - Какие версии можно удалить (совместимые).
   - Какие версии нужно сохранить (несовместимые).

5. **Запрос подтверждения** - перед изменениями команда спрашивает: *"Do you want to deduplicate? (y/N)"*.

6. **Выполнение дедупликации**:
   - **Быстрый режим** - удаляет только совместимые дубликаты (в рамках одной major-версии).
   - **Полный режим** (`--full`) - удаляет все дубликаты, даже если major-версии отличаются (может привести к breaking changes).

7. **Обновление lock-файла** - после удаления обновляет `mip-lock.yml`.

8. **Отчёт о результатах** - показывает сколько пакетов удалено и сколько осталось.

## Типичные ошибки

::: callout warning "❌ No duplicates found" icon:info
**Причина:** В проекте уже оптимизированное дерево зависимостей.

**Решение:** Это не ошибка - всё в порядке. Просто продолжайте работу.
:::

::: callout warning "❌ Lockfile not found" icon:alert-triangle
**Причина:** Отсутствует `mip-lock.yml`.

**Решение:**
1. Запустите `mip install` для создания lock-файла.
2. Затем повторите `mip dedupe`.
:::

::: callout tip "💡 Когда использовать dedupe" icon:lightbulb
- **После обновления** большого количества пакетов.
- **Перед сборкой** production-образа (чтобы уменьшить его размер).
- **При подозрении** на конфликты версий в рантайме.

Рекомендуется запускать `mip dedupe` перед коммитом, чтобы поддерживать проект в чистоте и сократить время CI/CD.
:::

::: collapsible "🧠 Что такое дедупликация?"
В менеджерах пакетов дублирование возникает, когда несколько пакетов требуют разные версии одной зависимости. Например, если пакет `A` требует `lodash@4.17.19`, а пакет `B` требует `lodash@4.17.21` - MIP установит обе версии.

**Быстрый режим** (`dedupe` без флагов):
- Проверяет только совместимые версии (в рамках одной major-версии).
- Удаляет дубликаты, которые можно безопасно заменить на одну общую версию.
- Работает быстро.

**Полный режим** (`dedupe --full`):
- Сканирует всё дерево зависимостей.
- Перестраивает его для максимального использования общих версий.
- Может занять больше времени, но даёт лучший результат.

**Важно:** Дедупликация никогда не нарушает требования пакетов - версия заменяется только если она совместима со всеми родительскими пакетами.
:::
