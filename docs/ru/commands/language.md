# mip language

::: callout info "Что это?" icon:globe
`mip language` - это команда для управления языком интерфейса MIP в рамках текущего проекта. MIP поддерживает мультиязычность и автоматически подстраивает вывод сообщений под язык, указанный в `mip.json`. Это удобно для команд с разными предпочтениями и для проектов, где важна локализация.
:::

## Синтаксис

```bash
mip language [lang]
```

## Аргументы

| Имя | Тип | Обязательный | Описание |
|-----|-----|--------------|----------|
| `lang` | строка | ❌ | Код языка (например, `en`, `ru`, `de`). Если не указан - показывает текущий язык. |

## Доступные языки

| Код | Язык |
|-----|------|
| `en` | English (по умолчанию) |
| `ru` | Русский |
| `de` | Deutsch |

## Логика работы

1. **Определение текущего языка** - MIP проверяет настройки в следующем порядке:
   - Язык из `mip.json` (поле `language`).
   - Язык системы (если не указан в проекте).
   - `en` (если не удалось определить).

2. **Показ текущего языка** - если `lang` не указан, команда выводит текущий язык и список доступных.

3. **Установка нового языка** - если указан код языка:
   - Проверяется, поддерживается ли язык.
   - Обновляется `mip.json` (добавляется или обновляется поле `language`).
   - Выводится подтверждение об успешной смене языка.

## Примеры

::: tabs
== tab "Просмотр текущего языка"
```bash
mip language
```

**Пример вывода:**
```
🌐 Current language: en
📋 Available languages: en, ru, de
```
:::
== tab "Смена языка на русский"
```bash
mip language ru
```

**Пример вывода:**
```
✅ Language set to: ru
```

После этого все команды MIP будут выводить сообщения на русском языке.
:::
== tab "Смена языка на немецкий"
```bash
mip language de
```

**Пример вывода:**
```
✅ Language set to: de
```
:::
== tab "Смена на неподдерживаемый язык"
```bash
mip language fr
```

**Пример вывода:**
```
❌ Invalid language: fr
📋 Available languages: en, ru, de
```
:::

## Типичные ошибки

::: callout warning "❌ Invalid language" icon:alert-triangle
**Причина:** Указан код языка, который не поддерживается.

**Решение:**
1. Используйте один из доступных языков: `en`, `ru`, `de`.
2. Проверьте правильность написания кода языка.
:::

::: callout info "💡 Совет" icon:lightbulb
- Язык сохраняется **в проекте** (в `mip.json`), а не глобально. Это позволяет разным проектам использовать разные языки.
- Если вы хотите изменить язык глобально - измените настройки системы или переменную окружения `LANG`.
- Для новых проектов язык по умолчанию определяется из системных настроек.
:::

## Связанные команды

- [mip init](/commands/init) - создаёт новый проект с указанием языка в `mip.json`.
- [mip info](/commands/info) - выводит информацию о пакетах на выбранном языке.

::: collapsible "🧠 Как работает мультиязычность в MIP?"
MIP использует **двухуровневую систему** локализации:

1. **Уровень проекта** - язык хранится в `mip.json` (поле `language`). Этот язык используется для всех команд в рамках проекта.

2. **Уровень системы** - если в проекте не указан язык, MIP смотрит на системную переменную `LANG` или `LC_ALL`.

**Приоритет:**
```
Язык в mip.json > Системная переменная LANG > en (по умолчанию)
```

**Переводы:**
Все сообщения MIP хранятся в файлах локализации:
```
lib/i18n/
├── en.json  # Английский
├── ru.json  # Русский
└── de.json  # Немецкий
```

Каждый файл содержит пары ключ-значение для всех сообщений команд:
```json
{
  "commands.install.installing": "Installing {package}...",
  "commands.install.installed": "✅ {package} installed"
}
```

При смене языка через `mip language` обновляется только поле в `mip.json`, а система подхватывает новый язык при следующем вызове команд.
:::

::: collapsible "🔧 Добавление новых языков"
Если вы хотите добавить поддержку нового языка:

1. Создайте файл в `lib/i18n/<lang>.json`.
2. Скопируйте структуру из существующего файла.
3. Переведите все сообщения на новый язык.
4. Добавьте код языка в массив `languages` в `lib/i18n/index.js`.
5. Готово! Теперь язык доступен через `mip language <lang>`.

**Важно:** При добавлении нового языка убедитесь, что все сообщения переведены. Отсутствующие переводы будут отображаться на английском (fallback).
:::
