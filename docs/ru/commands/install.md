# mip install

::: callout info "Что это?" icon:package
`mip install` - основная команда для установки зависимостей. Она скачивает пакеты из реестра npm, сохраняет их в глобальный кэш `~/.mip/store/`, создаёт симлинки в `node_modules/`, обновляет манифест и lock-файл. Поддерживает установку нескольких пакетов, глобальную установку и неинтерактивный режим для CI.
:::

## Синтаксис

```bash
mip install [packages...] [options]
```

## Аргументы

| Имя | Тип | Обязательный | Описание |
|-----|-----|--------------|----------|
| `packages` | массив | ❌ | Список пакетов для установки. Если не указан - устанавливаются все зависимости из `mip.yml`. |

## Опции

| Флаг | Описание |
|------|----------|
| `--save-dev` / `-D` | Сохраняет пакет в `devDependencies` |
| `--force` / `-f` | Принудительно переустанавливает пакеты (перезаписывает кэш) |
| `--global` / `-g` | Устанавливает пакет глобально |
| `--no-save` | Устанавливает пакет без сохранения в `mip.yml` |
| `--ci` | Неинтерактивный режим (автоматически пропускает peer-зависимости) |

## Логика работы

### `mip install` (без аргументов)
1. **Чтение `mip.yml`** - загружает зависимости из `dependencies` и `devDependencies`.
2. **Проверка lock-файла** - если `mip-lock.yml` отсутствует, он будет создан.
3. **Установка всех пакетов** - последовательно устанавливает все зависимости.
4. **Прогресс-бар** - отображает прогресс установки.
5. **Разрешение peer-зависимостей** - в интерактивном режиме спрашивает подтверждение, в `--ci` автоматически пропускает.

### `mip install <package>`
1. **Определение версии** - если указана `package@version`, устанавливается конкретная версия, иначе - `latest`.
2. **Резолвинг зависимостей** - получает информацию о пакете из реестра.
3. **Загрузка и распаковка** - скачивает tarball и распаковывает в глобальный кэш `~/.mip/store/`.
4. **Создание симлинка** - создаёт символическую ссылку в `node_modules/` на глобальный кэш.
5. **Установка зависимостей пакета** - рекурсивно устанавливает вложенные зависимости.
6. **Обновление lock-файла** - фиксирует точные версии в `mip-lock.yml`.
7. **Сохранение в `mip.yml`** - добавляет пакет в `dependencies` (если не указан `--no-save`).
8. **Обновление манифеста** - записывает информацию в `.mip/manifest.json`.

## Примеры

::: tabs
== tab "Установка всех зависимостей"
```bash
mip install
```

**Вывод:**
```
📦 Installing all dependencies (42 packages)...

  ████████████████████████████████████ 100% 42/42

✅ Installed 42 packages in 12.3s
```
:::
== tab "Установка нескольких пакетов"
```bash
mip install react electron
```

Устанавливает оба пакета параллельно.
:::
== tab "Установка конкретного пакета"
```bash
mip install express
```
:::
== tab "Установка конкретной версии"
```bash
mip install lodash@4.17.21
```
:::
== tab "Установка с сохранением в devDependencies"
```bash
mip install jest --save-dev
```
:::
== tab "CI-режим (без интерактива)"
```bash
mip install --ci
```

Автоматически пропускает все вопросы о peer-зависимостях.
:::
== tab "Глобальная установка"
```bash
mip install --global typescript
```
:::

## Глобальный кэш

MIP использует **глобальный кэш** в `~/.mip/store/`. Все пакеты хранятся один раз и используются всеми проектами:

```
~/.mip/store/
├── lodash/
│   └── 4.17.21/
│       ├── package.json
│       └── index.js
└── express/
    └── 4.18.2/
        ├── package.json
        └── lib/
```

Это экономит до 90% дискового пространства.

## Типичные ошибки

::: callout warning "❌ Package not found" icon:alert-triangle
**Причина:** Указанный пакет не существует в реестре npm.

**Решение:**
1. Проверьте правильность имени пакета.
2. Используйте `mip search` для поиска.
:::

::: callout warning "❌ Version not found" icon:alert-circle
**Причина:** Указанной версии пакета не существует.

**Решение:**
1. Проверьте доступные версии: `mip info <package>`
:::

::: callout danger "⚠️ Peer dependency conflicts" icon:skull
**Причина:** Пакет требует версию другого пакета, которая конфликтует с уже установленной.

**Решение:**
- В интерактивном режиме MIP предложит варианты.
- В CI используйте `--ci` для автоматического пропуска.
- Вручную установите требуемую версию.
:::

::: callout tip "💡 Совет" icon:lightbulb
- Для чистых установок в CI используйте `mip ci` - он быстрее и надёжнее.
- Используйте `--force` только если уверены, что кэш повреждён.
- Для глобальных пакетов используйте `mip global install` (более удобный интерфейс).
:::
