# mip update

::: callout info "Что это?" icon:arrow-up
`mip update` - это команда для обновления всех пакетов в проекте до последних доступных версий. Она проверяет каждую зависимость из `mip.yml`, показывает список пакетов, которые можно обновить, и после подтверждения устанавливает новые версии. Это удобный способ поддерживать проект в актуальном состоянии без ручного редактирования `mip.yml`.
:::

## Синтаксис

```bash
mip update
```

## Логика работы

1. **Чтение `mip.yml`** - загружает все зависимости из `dependencies` и `devDependencies`.

2. **Параллельная проверка** - для каждого пакета выполняется запрос к реестру для получения последней версии.

3. **Сравнение версий** - если последняя версия отличается от текущей, пакет попадает в список обновлений.

4. **Проверка глобального кэша** - показывает, будет ли пакет взят из глобального кэша (`🌍`) или скачан заново (`📡`).

5. **Проверка манифеста** - показывает, установлен ли пакет в проекте (`📋`) или нет (`❌`).

6. **Отображение списка** - показывает все пакеты, которые можно обновить, с текущей и новой версиями, а также статусом в кэше и манифесте.

7. **Запрос подтверждения** - спрашивает пользователя: *"Update these packages? (y/N)"*.

8. **Установка обновлений** - при подтверждении устанавливает новые версии с принудительной перезагрузкой (`force: true`).

9. **Обновление lock-файла** - автоматически обновляет `mip-lock.yml` с новыми версиями.

## Пояснение к иконкам

| Иконка | Значение |
|--------|----------|
| 🌍 | Пакет уже есть в глобальном кэше |
| 📡 | Пакет будет скачан из реестра |
| 📋 | Пакет установлен в проекте (есть в манифесте) |
| ❌ | Пакет отсутствует в манифесте |

## Типичные ошибки

::: callout warning "❌ No mip.yml found" icon:alert-triangle
**Причина:** Вы находитесь не в корне MIP-проекта.

**Решение:**
1. Перейдите в папку с проектом: `cd /path/to/project`
2. Убедитесь, что там есть `mip.yml`.
3. Если нет - создайте его: `mip init`
:::

::: callout warning "⚠️ Check failed for package" icon:alert-circle
**Причина:** Не удалось получить информацию о пакете из реестра.

**Решение:**
1. Проверьте интернет-соединение.
2. Проверьте правильность имени пакета в `mip.yml`.
3. Попробуйте позже.
:::

::: callout tip "💡 Совет" icon:lightbulb
`mip update` - это безопасный способ обновления:
- Показывает все изменения перед установкой.
- Не обновляет пакеты без вашего согласия.
- Автоматически обновляет lock-файл.

**После обновления:**
1. Проверьте, что проект работает: `mip run test`
2. Если есть проблемы - используйте `mip install` для переустановки.
3. Закоммитьте обновлённые `mip.yml` и `mip-lock.yml`.
:::

## Связанные команды

- [mip outdated](/ru/commands/outdated) - проверка устаревших пакетов (без установки).
- [mip install](/ru/commands/install) - установка конкретного пакета.
- [mip audit](/ru/commands/audit) - проверка безопасности.

::: collapsible "🧠 Как работает обновление?"
`mip update` выполняет обновление через уже существующий механизм установки:

**Этапы:**
1. **Проверка версий** - для каждого пакета получает последнюю версию из реестра.
2. **Сравнение** - если последняя версия не равна текущей, пакет добавляется в список.
3. **Установка** - вызывает `installPackage()` с флагом `force: true`, который:
   - Перезаписывает существующий пакет в глобальном кэше.
   - Обновляет `mip-lock.yml`.
   - Не меняет `mip.yml` (диапазон версий остаётся тем же).

**Важно:** `mip update` обновляет пакеты **до последней версии**, независимо от диапазона в `mip.yml`. Если вы хотите обновить только в рамках диапазона - используйте `mip install` с конкретной версией.

**Пример:**
```yaml
# mip.yml
dependencies:
  lodash: ^4.17.0  # диапазон разрешает 4.17.x
```
- `mip update` установит `4.17.21` (последняя в диапазоне).
- `mip install lodash` установит `4.17.21` (если нет lock-файла) или `4.17.19` (из lock-файла).
:::

::: collapsible "🔍 Сравнение update vs install"
| Команда | Что делает | Обновляет `mip.yml` | Обновляет `mip-lock.yml` | Подтверждение |
|---------|------------|---------------------|--------------------------|---------------|
| `mip update` | Обновляет все пакеты до последних версий | ❌ | ✅ | ✅ Запрашивает |
| `mip install` | Устанавливает пакеты (из lock-файла) | ❌ | ✅ | ❌ Нет |
| `mip install <pkg>` | Устанавливает конкретный пакет | ✅ (если нет `--no-save`) | ✅ | ❌ Нет |

**Рекомендация:**
- Используйте `mip outdated` для проверки.
- Затем `mip update` для обновления с подтверждением.
- Для CI используйте `mip ci` (без обновлений).
:::
