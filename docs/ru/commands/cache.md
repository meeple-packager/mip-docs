# mip cache

::: callout info "Что это?" icon:database
`mip cache` управляет кэшем загруженных пакетов. MIP хранит скачанные пакеты в двух местах: **локальный кэш** (`.mip/` в папке проекта) и **глобальный кэш** (`~/.mip/store/`). Команда позволяет посмотреть их размер, очистить или посмотреть, какие пакеты используются.
:::

## Синтаксис

```bash
mip cache <action> [--global]
```

## Аргументы

| Имя | Тип | Обязательный | Описание |
|-----|-----|--------------|----------|
| `action` | строка | ✅ | Действие с кэшем: `size`, `clean` или `usage` |

## Опции

| Флаг | Описание |
|------|----------|
| `--global` | Применяет действие к глобальному кэшу (`~/.mip/store/`) |

## Доступные действия

| Действие | Описание |
|----------|----------|
| `size` | Показывает размер кэша (локального или глобального) и количество пакетов в манифесте |
| `clean` | Полностью удаляет кэш, освобождая место |
| `usage` | Показывает, какие пакеты из глобального кэша используются в текущем проекте |

## Логика работы

### `size`
1. **Локальный кэш** — сканирует папку `.mip/` в текущем проекте и показывает её размер.
2. **Глобальный кэш** (с `--global`) — сканирует `~/.mip/store/` и показывает общий размер всех пакетов.
3. **Манифест** — показывает количество пакетов, установленных в текущем проекте.

### `clean`
1. **Локальный кэш** — удаляет папку `.mip/` в текущем проекте.
2. **Глобальный кэш** (с `--global`) — удаляет все пакеты из `~/.mip/store/`.

### `usage`
1. Сканирует глобальный кэш (`~/.mip/store/`).
2. Сравнивает с манифестом текущего проекта.
3. Показывает, какие пакеты используются (📋) и какие можно безопасно удалить (💀).

## Примеры

::: tabs
== tab "Просмотр локального кэша"
```bash
mip cache size
```

**Пример вывода:**
```
📦 Manifest: 3 packages
📦 Cache size: 42.50 MB
```
:::
== tab "Просмотр глобального кэша"
```bash
mip cache size --global
```

**Пример вывода:**
```
🌍 Global cache size: 156.30 MB
```
:::
== tab "Очистка локального кэша"
```bash
mip cache clean
```

**Пример вывода:**
```
🧹 Cache cleaned: 42.50 MB freed
```
:::
== tab "Очистка глобального кэша"
```bash
mip cache clean --global
```

**Пример вывода:**
```
🌍 Global cache cleaned: 156.30 MB freed
```
:::
== tab "Просмотр использования глобального кэша"
```bash
mip cache usage --global
```

**Пример вывода:**
```
📊 Global cache usage
────────────────────────────────────────────────────────────
  📋 lodash@4.17.21  21.50 MB
  📋 express@4.18.2  25.00 MB
  💀 jest@29.7.0    32.00 MB
────────────────────────────────────────────────────────────
📦 Total: 78.50 MB

Legend:
  📋 - used in current project
  💀 - not used (can be cleaned)
```
:::

## Типичные ошибки

::: callout warning "⚠️ No lockfile or manifest found" icon:alert-triangle
**Причина:** В проекте нет установленных пакетов.

**Решение:** Это не ошибка — просто нечего очищать. Установите пакеты через `mip install`.
:::

::: callout warning "⚠️ No global cache found" icon:alert-circle
**Причина:** Глобальный кэш ещё не создан (ни один пакет не был установлен).

**Решение:** Начните устанавливать пакеты — кэш появится автоматически.
:::

::: callout tip "💡 Совет" icon:lightbulb
Очищайте кэш если:
- У вас заканчивается место на диске.
- Вы подозреваете повреждение кэша (странные ошибки при установке).
- Вы переключились между проектами с разными версиями пакетов.

Используйте `mip cache usage --global` для анализа, какие пакеты можно безопасно удалить.
:::

## Где хранится кэш

| Тип | Путь | Назначение |
|-----|------|------------|
| Локальный | `.mip/` | Кэш для текущего проекта (tarball'ы и временные файлы) |
| Глобальный | `~/.mip/store/` | Хранилище всех версий пакетов для всех проектов |

## Связанные команды

- [mip install](/ru/commands/install) — устанавливает пакеты и создаёт кэш.
- [mip doctor](/ru/commands/doctor) — проверяет состояние проекта, включая целостность кэша.

::: collapsible "🧠 Почему два кэша?"
MIP использует **два уровня кэширования** для баланса между скоростью и дисковым пространством:

1. **Глобальный кэш** (`~/.mip/store/`) — хранит все версии всех пакетов, которые вы когда-либо устанавливали в любых проектах. Это экономит трафик и место — если пакет уже есть глобально, MIP просто создаёт симлинк в проекте.

2. **Локальный кэш** (`.mip/` в проекте) — хранит только tarball'ы для быстрой переустановки и временные файлы. Он не дублирует пакеты.

Очистка глобального кэша удаляет пакеты из `~/.mip/store/`. При следующей установке они будут скачаны заново, но это освободит много места.
:::
