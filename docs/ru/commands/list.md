# mip list

::: callout info "Что это?" icon:list
`mip list` — это команда для просмотра всех пакетов, установленных в текущем проекте. Она показывает пакеты из манифеста `.mip/manifest.json`, их версии и пути в глобальном кэше. Это удобно для быстрой проверки состояния зависимостей.
:::

## Синтаксис

```bash
mip list
```

## Логика работы

1. **Проверка `mip.yml`** — если файл отсутствует, команда выводит сообщение об ошибке.

2. **Чтение манифеста** — из `.mip/manifest.json` извлекаются все установленные пакеты.

3. **Определение источника** — для каждого пакета показывается:
   - 🌍 — пакет из глобального кэша `~/.mip/store/`
   - 🔗 — пакет, связанный через `mip link`
   - 📁 — локальный пакет (для обратной совместимости)

4. **Вывод версий и путей** — для каждого пакета показывается версия и путь в глобальном кэше.

5. **Итоговый подсчёт** — общее количество пакетов.

## Примеры

::: tabs
== tab "Просмотр установленных пакетов"
```bash
mip list
```

**Пример вывода:**
```

  🌍 express@4.18.2
     /home/user/.mip/store/express/4.18.2
  🌍 lodash@4.17.21
     /home/user/.mip/store/lodash/4.17.21
  🌍 jest@29.7.0
     /home/user/.mip/store/jest/29.7.0

📊 Total: 3 package(s)
```
:::
== tab "Пустой проект (без пакетов)"
```bash
mip list
```

**Пример вывода:**
```
📦 Installed packages:

  (no packages installed)

📊 Total: 0 package(s)
```
:::
== tab "Связанные пакеты"
```bash
mip list
```

**Пример вывода:**
```
📦 Installed packages:

  🔗 my-lib@1.0.0
     /home/user/dev/my-lib
  🌍 lodash@4.17.21
     /home/user/.mip/store/lodash/4.17.21

📊 Total: 2 package(s)
```
:::

## Что означают иконки

| Иконка | Значение |
|--------|----------|
| 🌍 | Пакет из глобального кэша (`~/.mip/store/`) |
| 🔗 | Локально связанный пакет (через `mip link`) |
| 📁 | Локальный пакет (обратная совместимость) |

## Типичные ошибки

::: callout warning "❌ No mip.yml found" icon:alert-triangle
**Причина:** Вы находитесь не в корне MIP-проекта.

**Решение:**
1. Перейдите в папку с проектом: `cd /path/to/project`
2. Убедитесь, что там есть `mip.yml`.
3. Если нет — создайте его: `mip init`
:::

::: callout tip "💡 Совет" icon:lightbulb
`mip list` — это первая команда для диагностики:
- Проверьте, что все пакеты установлены.
- Убедитесь, что пути в манифесте корректны.
- Просмотрите версии пакетов перед обновлением.

Используйте `mip list` перед `mip outdated`, чтобы понять, что у вас установлено.
:::
