# Troubleshooting (Users)

## `mip.json` не найден
Симптом: команды `run/list/update/ci` пишут, что `mip.json` отсутствует.

Причина:
- вы запускаете `mip` не в корне проекта, где лежит `mip.json`.

Решение:
- перейдите в папку проекта:
```bash
cd /path/to/project
mip list
```

## `mip ci`: нет `mip-lock.json`
Симптом: `mip-lock.json` не найден.

Решение:
- выполните обычную установку, чтобы появился lockfile:
```bash
mip install <pkg>
# или
mip install
```

## `mip exec <cmd>` не находит бинарник
По реализации `mip exec` делает:
1) ищет `./.mip/<pkg>/<version>/node_modules/.bin/<cmd>`
2) ищет `./.mip/<pkg>/<version>/.bin/<cmd>`
3) если не нашёл — запускает системную команду с таким именем.

Если ваш CLI не найден:
- проверьте, что пакет действительно установился в `.mip/`
- проверьте, что внутри пакета есть `.bin/<cmd>` (или `node_modules/.bin/<cmd>`)

## `mip run` не запускает скрипт
Симптом: `script not found` / нет `scripts.<name>`.

Причина:
- в `mip.json` нет нужного ключа в `scripts`.

Решение:
- добавьте:
```json
{
  "scripts": {
    "start": "node app.js"
  }
}
```
и затем:
```bash
mip run start
```

## `mip audit` не работает из-за сети
В реализации `audit` при проблемах с сетью может выводить ошибку/пропускать скан.

Решение:
- проверьте доступ в интернет к `registry.npmjs.org`
- затем повторите:
```bash
mip audit
```

## Windows: проблемы с путями/аргументами
`mip run` запускает строку через shell (а не split на args), поэтому корректнее обрабатывает кавычки/пробелы.

Если команда ведёт себя странно:
- проверьте кавычки в строке скрипта `mip.json` (особенно пути с пробелами)
