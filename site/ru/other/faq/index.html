Расширенный FAQ ❓

Собрал ответы на вопросы, которые возникают у опытных пользователей и разработчиков.

📦 Общие вопросы

Чем MIP отличается от npm?

MIP не дублирует пакеты в node_modules. Вместо этого все пакеты хранятся в глобальном кэше ~/.mip/store/. Это экономит место и ускоряет установку.

Сравнение:

Характеристика npm MIP
Дублирование пакетов ✅ Есть ❌ Нет
Глобальный кэш ❌ Нет ✅ Есть
Работа офлайн ❌ Нет ✅ Да (после первой установки)
Плагины ❌ Нет ✅ Есть
YAML-конфиги ❌ Нет ✅ Есть

Где хранятся пакеты?

В глобальном кэше ~/.mip/store/:

~/.mip/store/
├── lodash/
│   └── 4.17.21/
│       ├── package.json
│       └── index.js
└── express/
    └── 4.18.2/
        ├── package.json
        └── lib/

В проекте только симлинки в node_modules/.

Можно ли удалить ~/.mip/?

Да, но тогда пакеты придётся переустанавливать заново.

🔌 Плагины

Как создать свой плагин?

mip plugin create my-plugin

Плагин появится в plugins/my-plugin/.

Где хранятся скомпилированные плагины?

В ~/.mip_cache/plugins/.

Как выполнить команду плагина?

mip pe my-plugin my-command

Можно ли передавать аргументы в команду плагина?

Да:

mip pe my-plugin my-command arg1 arg2

Какие хуки доступны в плагинах?

Все хуки описаны в разделе Хуки и плагины. Основные:

Хук Когда вызывается
beforeInstall Перед установкой
afterInstall После установки
onError При ошибке

Можно ли использовать API MIP в плагинах?

Да. В init() плагин получает объект api:

init: async ({ api }) => {
  const config = api.getMipJson();
  const version = api.getVersion();
  console.log(`MIP v${version}`);
}

🐛 Отладка и ошибки

Как включить debug-режим?

DEBUG=1 mip install lodash

Где посмотреть логи?

Логи выводятся в консоль. Для плагинов-логгеров можно писать в файл.

Команда зависла. Что делать?

Нажми Ctrl+C. Если зависло на npm install в плагине — проверь package.json плагина.

Почему mip pe не видит мою команду?

  1. Проверь, что плагин активирован: mip plugin list
  2. Проверь, что команда зарегистрирована в commands: {}
  3. Перекомпилируй плагин: mip plugin compile my-plugin

🚀 Производительность

Почему MIP быстрее npm?

  • Параллельная загрузка (до 10 пакетов одновременно)
  • Трёхуровневое кэширование (SuperCache)
  • HTTP/2
  • Глобальный кэш (не нужно качать повторно)

Бенчмарки:

npm:  4.2s  (холодный кэш)
pnpm: 3.1s
mip:  2.8s  (холодный кэш)
mip:  0.4s  (тёплый кэш) ← в 10 раз быстрее npm

Как очистить кэш?

# Локальный кэш
mip cache clean

# Глобальный кэш
mip cache clean --global

Как посмотреть размер кэша?

mip cache size
mip cache size --global

🔧 Настройка

Где лежит конфиг MIP?

В mip.yml в корне проекта.

Как сменить язык?

mip language ru

Какие языки поддерживаются?

en, ru, es, fr, de, it, pt, zh, ja, ko.

Как мигрировать со старого mip.json?

При первом запуске MIP автоматически мигрирует:

  • mip.json → mip.yml (с бэкапом)
  • mip-lock.json → mip-lock.yml (с бэкапом)

🧪 CI/CD

Как использовать MIP в CI?

mip ci

Что делает mip ci?

Устанавливает пакеты строго по mip-lock.yml, игнорируя mip.yml.

Что такое frozen-lockfile?

mip ci --frozen-lockfile

Проверяет, что lock-файл синхронизирован с mip.yml. Если есть расхождения — завершается с ошибкой.

Как проверить безопасность в CI?

mip audit

🧩 Работа с пакетами

Как установить пакет без сохранения в mip.yml?

mip install lodash --no-save

Как установить конкретную версию?

mip install lodash@4.17.21

Как обновить все пакеты?

mip update

Как удалить пакет?

mip uninstall lodash

Как посмотреть установленные пакеты?

mip list

Как узнать, почему установлен пакет?

mip why lodash

🛠️ Разработка

Как добавить свою команду в MIP?

Два способа:

  1. Встроенная команда (fork): править bin/mip.js
  2. Плагин: через api.registerCommand()

Подробнее: Добавление команд.

Как протестировать плагин?

mip plugin compile my-plugin
mip plugin activate my-plugin
mip pe my-plugin my-command
