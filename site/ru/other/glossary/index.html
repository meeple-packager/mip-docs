Глоссарий 📚

Собрал основные термины, которые встречаются в MIP и пакетных менеджерах.

🔤 A–Z

API

Application Programming Interface — набор методов, доступных плагинам для взаимодействия с MIP.

Audit

Проверка безопасности зависимостей. MIP проверяет пакеты на известные уязвимости через npm advisory database.

🅱️ B

Bin

Исполняемые файлы, которые добавляются в PATH при установке пакета. В MIP находятся в ~/.mip/store/<name>/<version>/node_modules/.bin/.

Branch

Ветка в Git. MIP использует --branch в команде repo для просмотра конкретной ветки.

🅲️ C

Cache

Кэш — временное хранилище данных для ускорения повторных операций. В MIP: SuperCache (3 уровня: память, диск, in-flight), глобальный кэш ~/.mip/store/.

CI

Continuous Integration — непрерывная интеграция. Команда mip ci оптимизирована для CI-сред.

CLI

Command Line Interface — интерфейс командной строки. MIP — это CLI-инструмент.

CVSS

Common Vulnerability Scoring System — система оценки уязвимостей. Используется в mip audit.

🅳️ D

Dedupe

Дедупликация — удаление дублирующихся версий пакетов. Команда mip dedupe.

Dependency

Зависимость — пакет, который требуется другому пакету для работы.

Dependency Tree

Дерево зависимостей — граф всех зависимостей пакета, включая транзитивные.

Dev Dependency

Зависимость, необходимая только для разработки (тесты, сборка). Хранится в devDependencies в mip.yml.

🅴️ E

Exec

Выполнение бинарника из установленного пакета. Команда mip exec.

Exports

Секция в package.json, которая определяет публичные пути пакета.

🅵️ F

Fallback

Запасной вариант. Например, LegacyFallback в MIP.

Frozen Lockfile

Lock-файл в замороженном состоянии. При mip ci --frozen-lockfile установка прерывается, если lock-файл не синхронизирован с mip.yml.

🅶️ G

Global Install

Глобальная установка — пакет становится доступен из любого места. В MIP: mip global install.

Global Store

Глобальное хранилище пакетов в ~/.mip/store/. Все пакеты хранятся один раз и используются всеми проектами.

GitHub API

API для работы с GitHub. Используется в команде repo.

🅷️ H

Hook

Хук — место в коде, где плагины могут выполнять свою логику. В MIP: beforeInstall, afterInstall, onError и др.

HTTP/2

Протокол для ускорения загрузки. MIP использует HTTP/2 для параллельных загрузок пакетов.

🅸️ I

i18n

Internationalization — интернационализация. В MIP: поддержка 10 языков.

Integrity

Целостность пакета. В mip-lock.yml хранятся SHA256-хеши для проверки.

Install

Установка пакета. Команда mip install.

🅻️ L

Legacy

Устаревший пакет. В MIP: mip legacy для работы с ними.

Lockfile

Файл с зафиксированными версиями пакетов. В MIP: mip-lock.yml.

LRU Cache

Кэш, который удаляет старые элементы при переполнении. Используется в SuperCache.

🅼️ M

MIP

MInimal Package Manager — минималистичный менеджер пакетов.

MIP Cache

Кэш MIP — лежит в .mip/cache/ (локальный) и ~/.mip/store/ (глобальный).

MIP Lock

Lock-файл — mip-lock.yml.

MIP Store

Глобальное хранилище пакетов — ~/.mip/store/.

Manifest

Манифест — .mip/manifest.json, хранит информацию об установленных пакетах.

Monorepo

Монорепозиторий — проект с несколькими пакетами в одном репозитории. В MIP: mip workspaces.

🅽️ N

Node.js

Среда выполнения JavaScript. MIP написан на Node.js.

Node Modules

Папка с установленными пакетами. В MIP вместо node_modules используются симлинки на глобальный кэш.

npm

Node Package Manager — оригинальный менеджер пакетов для Node.js.

🅾️ O

Options

Опции — флаги командной строки. Например, --save-dev, --force.

Outdated

Устаревший пакет (есть новая версия). Команда mip outdated.

🅿️ P

Package

Пакет — единица распространения кода в MIP.

Package.json

Файл с метаданными пакета. MIP поддерживает его для совместимости с npm.

Parallel Download

Параллельная загрузка — скачивание нескольких пакетов одновременно.

PATH

Переменная окружения, в которой перечислены папки с исполняемыми файлами.

Peer Dependency

Зависимость, которую пакет ожидает от родителя. В MIP: PeerResolver.

Plugin

Плагин — расширение функциональности MIP.

Plugin Manager

Менеджер плагинов — управляет загрузкой и регистрацией плагинов.

🆁️ R

Registry

Реестр пакетов — например, registry.npmjs.org.

Resolved

URL tarball-архива пакета, зафиксированный в lock-файле.

Resolver

Компонент, который разрешает версии зависимостей.

🆂️ S

Semver

Semantic Versioning — семантическое версионирование (MAJOR.MINOR.PATCH).

Sidebar

Боковая панель в документации.

SPA

Single Page Application — одно страничное приложение. Документация MIP работает в SPA-режиме.

Stream Extract

Потоковая распаковка — распаковка tarball без сохранения на диск.

Super Cache

Трёхуровневый кэш в MIP (память → диск → in-flight).

Symlink

Символическая ссылка. MIP использует их для связи node_modules/ с глобальным кэшем.

🆃️ T

Tarball

Архив .tgz с пакетом.

TUI

Text User Interface — текстовый интерфейс. Например, mip repo в TUI-режиме.

🆄️ U

Uninstall

Удаление пакета. Команда mip uninstall.

Update

Обновление пакетов. Команда mip update.

🆅️ V

Vulnerability

Уязвимость — ошибка в пакете, которая может быть использована злоумышленником.

🆆️ W

Workspace

Воркспейс — отдельный пакет в монорепозитории. Команда mip workspaces.

Why

Почему установлен пакет? Команда mip why.

❓ Вопросы?

Если встретили термин, которого нет в глоссарии — сообщите. Дополним! 📝
