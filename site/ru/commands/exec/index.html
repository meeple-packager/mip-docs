mip exec — выполнение бинарников из зависимостей
  1. Команды
  2. exec

mip exec

Что это?

mip exec — это умный запускатор бинарников из ваших зависимостей. Вместо того чтобы вручную искать путь к исполняемому файлу в node_modules/.bin, вы просто передаёте имя команды, а MIP сам находит её в манифесте и выполняет. Это особенно удобно для скриптов, тестовых раннеров и CLI-утилит, установленных локально.

Синтаксис

mip exec <command> [args...]

Аргументы

Имя Тип Обязательный Описание
command строка ✅ Имя исполняемого файла (например, jest, eslint, nodemon)
args массив ❌ Аргументы, передаваемые в исполняемый файл

Логика работы

  1. Поиск в манифесте — команда читает .mip/manifest.json и ищет установленный пакет, содержащий бинарник с указанным именем.

  2. Поиск в локальном кэше — если не найдено в манифесте, проверяет .mip/packages/ (обратная совместимость).

  3. Проверка системной команды — если бинарник не найден нигде, команда пытается выполнить его как системный.

  4. Передача аргументов — все аргументы после <command> передаются дочернему процессу.

  5. Интерактивный режим — stdio: 'inherit' позволяет взаимодействовать с командой в реальном времени.

Примеры

Запуск тестов через Jest
mip exec jest --coverage

Вывод:

🔧 Found in manifest (jest): jest@29.7.0/node_modules/.bin/jest

✨ Executing: jest --coverage

 PASS  src/utils.test.js
 PASS  src/api.test.js
-----------------------------|---------|----------|---------
...

== tab “Запуск ESLint”

mip exec eslint src/ --fix

Вывод:

🔧 Found in manifest (eslint): eslint@8.56.0/node_modules/.bin/eslint

✨ Executing: eslint src/ --fix

✅ All files fixed

:::
== tab “Использование системной команды”

mip exec ls -la

Вывод:

🔧 Using system command: ls

✨ Executing: ls -la

total 48
drwxr-xr-x  10 user  staff   320 Jan 15 14:23 ...

:::
== tab “Запуск nodemon с дополнительными аргументами”

mip exec nodemon --watch src app.js

Вывод:

🔧 Found in manifest (nodemon): nodemon@3.0.1/node_modules/.bin/nodemon

✨ Executing: nodemon --watch src app.js

[nodemon] watching src/
[nodemon] starting node app.js
Server running on port 3000

:::

Типичные ошибки

❌ Command not found

Причина: Исполняемый файл не найден ни в манифесте, ни в системе.

Решение:

  1. Убедитесь, что пакет установлен: mip list | grep <command>
  2. Проверьте правильность имени команды.
  3. Установите пакет, если он отсутствует: mip install <package>
❌ mip exec index.js

Причина: mip exec предназначен для запуска бинарников, а не скриптов.

Решение:
Для запуска скриптов используйте:

mip exec node index.js   # или
mip run start            # если скрипт добавлен в mip.yml
💡 Совет

mip exec — это более удобная альтернатива npx для локальных проектов:

  • Работает быстрее, так как не проверяет глобальные кэши.
  • Гарантирует использование той версии, которая установлена в проекте.
  • Не требует добавления скриптов в mip.yml.

Используйте mip exec для разовых запусков или в CI-скриптах.

Связанные команды

  • mip run — запускает скрипты из mip.yml.
  • mip install — устанавливает пакеты, бинарники которых становятся доступны через exec.
  • mip list — показывает установленные пакеты.
🔧 Как exec ищет бинарники?

Алгоритм поиска:

  1. Манифест (.mip/manifest.json):

    {
  "jest": {
    "version": "29.7.0",
    "path": "/home/user/.mip/store/jest/29.7.0"
  }
}

    Проверяет пути:

    • node_modules/.bin/<command>
    • .bin/<command>
    • bin/<command>

  2. Если бинарник найден — он выполняется.

  3. Если не найден — выполняется системная команда (например, ls, git).

Важно: Если у вас несколько версий одного пакета, будет выбрана первая найденная. Для точного контроля используйте mip exec <package> <command>.
