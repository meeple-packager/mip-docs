mip audit — проверка безопасности зависимостей
  1. Команды
  2. audit

mip audit

Что это?

mip audit сканирует все установленные пакеты и их зависимости, сверяя их с базой данных уязвимостей (GitHub Advisory Database, npm Security). Вы получаете отчёт о критических, высоких, средних и низких уязвимостях, а также рекомендации по обновлению.

Синтаксис

mip audit [--fix]

Опции

Флаг Описание
--fix Автоматически обновлять уязвимые пакеты до безопасных версий (если возможно)

Логика работы

  1. Чтение lock-файла — команда проверяет наличие mip-lock.yml в текущей директории. Если файл отсутствует — выполнение прекращается.

  2. Сканирование зависимостей — для каждого пакета из lock-файла отправляется запрос к реестру npm для получения списка известных уязвимостей.

  3. Фильтрация — проверяется, попадает ли текущая версия пакета под vulnerable_versions из базы уязвимостей.

  4. Сортировка — все найденные уязвимости сортируются по CVSS-оценке (от наиболее критичных к наименее опасным).

  5. Вывод отчёта — группировка по уровням серьёзности: 🔴 Critical, 🟠 High, 🟡 Moderate, 🔵 Low.

Если указан флаг --fix:
6. Поиск исправлений — для каждой уязвимости проверяется, существует ли patched_versions.
7. Обновление пакетов — выполняется mip install <pkg>@<fixed-version>.
8. Синхронизация mip.yml — автоматически обновляется mip.yml.

Примеры

Стандартная проверка
mip audit

Выводит список всех уязвимостей с указанием:

  • Название пакета и его версия
  • Уровень серьёзности и описание
  • CVSS-оценка
  • Доступные патчи или рекомендация обновить вручную
  • Ссылка на детальное описание уязвимости

Пример вывода:

== tab “Автоисправление”

mip audit --fix
  1. Находит все уязвимые пакеты.
  2. Для каждого проверяет наличие исправленной версии в рамках текущего major-релиза (чтобы избежать breaking changes).
  3. Автоматически обновляет пакет и синхронизирует mip.yml.
  4. Выводит отчёт о том, какие пакеты были обновлены.
    :::

Типичные ошибки

❌ ERR_NO_CONNECTION

Причина: Не удалось подключиться к реестру npm (проблемы с сетью или реестр недоступен).

Решение:

  1. Проверьте интернет-соединение.
  2. Убедитесь, что реестр npm доступен.
  3. Если вы используете прокси — настройте переменные окружения.
❌ lock_not_found

Причина: В текущей директории отсутствует файл mip-lock.yml.

Решение:

  1. Убедитесь, что вы находитесь в корне проекта с mip.yml.
  2. Запустите mip install, чтобы создать lock-файл.
  3. Попробуйте снова: mip audit.
⚠️ update_failed

Причина: Не удалось установить исправленную версию пакета (конфликты зависимостей или ошибка реестра).

Решение:

  1. Проверьте, существует ли указанная версия пакета в реестре.
  2. Попробуйте обновить пакет вручную: mip install <pkg>@<version>.
  3. Проверьте, нет ли конфликтов с другими зависимостями.
🔄 Автофикс только в рамках major-версии

mip audit --fix никогда не обновит пакет до версии с другим major-номером (например, с 1.x на 2.x). Это защищает от потенциальных breaking changes. Если безопасная версия требует смены major — команда предложит обновить вручную.

Связанные команды

  • mip outdated — показывает устаревшие пакеты (не только с уязвимостями).
  • mip update — обновляет все пакеты до последних версий.
  • mip doctor — проверяет общее состояние проекта (включая конфликты зависимостей).
🔧 Как работает поиск исправлений

При включении --fix команда:

  1. Проверяет, существует ли patched_versions.
  2. Парсит список доступных патчей.
  3. Выбирает минимальную версию, которая больше текущей в рамках того же major-релиза.
  4. Выполняет mip install <pkg>@<version>.
  5. Обновляет mip.yml.

Этот подход гарантирует минимальные изменения и безопасное обновление.
