mip server - веб-дашборд для MIP
  1. Команды
  2. server

mip server

Что это?

mip server запускает красивый веб-дашборд в реальном времени для вашего MIP-проекта. Он предоставляет визуальный обзор зависимостей проекта, глобального хранилища, настроенных реестров, доступных скриптов и системной информации - всё в одном месте с автоматическим обновлением.

Синтаксис

mip server <подкоманда> [опции]

Подкоманды

Подкоманда Описание
start [порт] Запустить веб-сервер (порт по умолчанию: 3000)
stop Остановить работающий сервер
status Показать статус сервера (запущен/остановлен)

Опции

Флаг Описание
--open, -o Автоматически открыть дашборд в браузере
--host Слушать на всех сетевых интерфейсах (0.0.0.0) вместо localhost

Возможности дашборда

📊 Вкладка «Обзор»

  • Статистика пакетов - общее количество установленных пакетов
  • Размер хранилища - использование глобального хранилища
  • Разбивка зависимостей - production vs development
  • Количество скриптов - доступные npm-скрипты
  • Количество реестров - настроенные реестры
  • Количество workspace - монорепозиторные workspace
  • Топ-5 пакетов - самые большие пакеты по размеру
  • Быстрый просмотр реестров - все настроенные реестры со статусом

📦 Вкладка «Пакеты»

  • Полный список всех установленных пакетов
  • Имя пакета, версия, размер и путь
  • Индикатор статуса (существует/отсутствует в глобальном хранилище)
  • Сортировка по размеру (сначала самые большие)

💾 Вкладка «Хранилище»

  • Общий размер глобального хранилища
  • Количество пакетов в хранилище
  • Путь к хранилищу
  • Топ-50 пакетов по размеру в глобальном кеше

🔗 Вкладка «Реестры»

  • Все настроенные реестры с URL
  • Статус токена (установлен/отсутствует)
  • Индикатор реестра по умолчанию
  • Встроенный реестр (npm) всегда отображается

📜 Вкладка «Скрипты»

  • Все npm-скрипты, определённые в mip.yml или mip.json
  • Имена скриптов и команды
  • Быстрый доступ к доступной автоматизации

⚙️ Вкладка «Система»

  • Версия Node.js
  • Версия MIP
  • Платформа и архитектура
  • Использование памяти (свободно/всего)
  • Количество ядер CPU
  • Время работы системы (uptime)
  • Средняя нагрузка (load average)
  • Текущая рабочая директория
  • Домашняя директория

Как это работает

  1. Процесс сервера - MIP запускает легковесный HTTP-сервер, используя встроенный модуль Node.js http.

  2. API эндпоинты - сервер предоставляет RESTful эндпоинты:

    • /api/project - конфигурация проекта
    • /api/packages - установленные пакеты
    • /api/store - информация о глобальном хранилище
    • /api/registries - конфигурация реестров
    • /api/system - системная информация
    • /api/health - проверка состояния

  3. Интерфейс дашборда - одностраничное приложение с вкладками, которое получает данные из API каждые 3 секунды.

  4. Управление процессами - PID сервера сохраняется в ~/.mip/server.pid для контроля процесса.

  5. Автообновление - дашборд автоматически обновляется каждые 3 секунды, поддерживая информацию актуальной.

Примеры

Запуск сервера
mip server start

Вывод:

🌐 MIP Server running at http://localhost:3000
📊 Dashboard: http://localhost:3000/
📁 Project: /home/user/my-project
🔧 Press Ctrl+C to stop

Откройте браузер и перейдите по адресу http://localhost:3000, чтобы увидеть дашборд.

== tab “Запуск на другом порту”

mip server start 8080

Запускает сервер на порту 8080 вместо порта по умолчанию 3000.
:::

== tab “Запуск с открытием браузера”

mip server start --open

Запускает сервер и автоматически открывает дашборд в вашем браузере по умолчанию.
:::

== tab “Запуск на всех интерфейсах”

mip server start --host

Позволяет получить доступ с других устройств в сети (полезно для демонстрации дашборда коллегам).
:::

== tab “Остановка сервера”

mip server stop

Вывод:

✅ Server stopped (PID: 12345)

:::

== tab “Проверка статуса”

mip server status

Вывод:

✅ Server is running (PID: 12345)
   To stop: mip server stop

Или если не запущен:

ℹ️ Server is not running

:::
:::

Частые ошибки

❌ ERR_PORT_IN_USE

Причина: Порт 3000 (или указанный порт) уже используется.

Решение:

  1. Используйте другой порт: mip server start 8080
  2. Остановите существующий сервер: mip server stop
  3. Или завершите процесс вручную, используя PID, указанный в сообщении об ошибке.
❌ ERR_SERVER_ALREADY_RUNNING

Причина: Сервер уже запущен.

Решение:

  1. Проверьте статус: mip server status
  2. Сначала остановите: mip server stop
  3. Затем запустите снова: mip server start
⚠️ ERR_NO_CONFIG

Причина: Не найден mip.yml или mip.json в текущей директории.

Решение:

  1. Убедитесь, что вы находитесь в директории MIP-проекта.
  2. Выполните mip init для создания нового проекта.
  3. Или перейдите в директорию проекта с файлом конфигурации.

Связанные команды

  • mip doctor - диагностика системы
  • mip list - список установленных пакетов (CLI)
  • mip config - управление конфигурацией
  • mip registry - управление реестрами
🔧 Как работает сервер под капотом

Сервер реализован в одном файле с использованием встроенного модуля Node.js http (без внешних зависимостей).

Поток выполнения:

  1. При вызове mip server start MIP:

    • Проверяет, не запущен ли уже сервер (используя ~/.mip/server.pid)
    • Создаёт HTTP-сервер
    • Начинает слушать указанный порт
    • Записывает PID в ~/.mip/server.pid
    • Опционально: открывает браузер

  2. Ответы API генерируются по запросу путём чтения:

    • Конфигурации проекта из mip.yml или mip.json
    • Манифеста пакетов из .mip/manifest.json
    • Глобального хранилища из ~/.mip/store/
    • Системной информации из модуля Node.js os

  3. Дашборд отдаётся как один HTML-файл со встроенными CSS и JavaScript. JavaScript:

    • Получает данные из API-эндпоинтов каждые 3 секунды
    • Отображает данные в интерфейсе с вкладками
    • Поддерживает горячие клавиши (Ctrl+R для обновления)

  4. Корректное завершение - сервер обрабатывает сигналы SIGINT и SIGTERM для очистки PID-файла.

💡 Совет: Поделитесь с командой

Используйте mip server start --host, чтобы разрешить другим устройствам в вашей сети доступ к дашборду. Сервер будет слушать на всех интерфейсах, и вы можете поделиться своим локальным IP-адресом с коллегами:

# Запуск на всех интерфейсах
mip server start --host

# Ваша команда может получить доступ по адресу:
# http://192.168.1.100:3000  (замените на ваш IP)
