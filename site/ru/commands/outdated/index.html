mip outdated — проверка устаревших пакетов
  1. Команды
  2. outdated

mip outdated

Что это?

mip outdated — это команда для проверки устаревших пакетов в вашем проекте. Она сравнивает версии пакетов, указанные в mip.yml, с последними доступными версиями в реестре npm. Команда показывает, какие пакеты можно обновить, и выводит результат в удобной таблице или в JSON для автоматизации.

Синтаксис

mip outdated [--json]

Опции

Флаг Описание
--json Выводит результат в формате JSON (удобно для CI/CD и скриптов)

Логика работы

  1. Чтение mip.yml — команда загружает зависимости из dependencies и devDependencies.

  2. Проверка актуальности — для каждого пакета выполняется запрос к реестру npm для получения последней версии.

  3. Сравнение версий — если версия из mip.yml отличается от последней в реестре, пакет считается устаревшим.

  4. Проверка глобального кэша — показывает, есть ли пакет в глобальном кэше (🌍) или его нужно скачать (📡).

  5. Проверка манифеста — показывает, установлен ли пакет в проекте (📋) или нет (❌).

  6. Вывод результата:

    • Обычный режим — таблица с колонками: Package, Current, Latest, Store, Manifest.
    • JSON режим (--json) — структурированный вывод для автоматизации.

  7. Код возврата — в JSON режиме возвращает код 1, если есть устаревшие пакеты (удобно для CI).

Пояснение к колонкам

Колонка Значение
Package Имя пакета
Current Текущая версия в проекте
Latest Последняя версия в реестре
Store 🌍 — есть в глобальном кэше, 📡 — будет скачан
Manifest 📋 — установлен в проекте, ❌ — отсутствует

Типичные ошибки

❌ No mip.yml found

Причина: Вы находитесь не в корне MIP-проекта.

Решение:

  1. Перейдите в папку с проектом: cd /path/to/project
  2. Убедитесь, что там есть mip.yml.
  3. Если нет — создайте его: mip init
❌ Package not found in registry

Причина: Пакет не найден в реестре npm (возможно, опечатка или пакет удалён).

Решение:

  1. Проверьте правильность имени пакета в mip.yml.
  2. Убедитесь, что пакет существует в реестре.
  3. Исправьте имя и повторите попытку.
💡 Совет

mip outdated — это первый шаг перед обновлением:

  1. Запустите mip outdated, чтобы увидеть, что нужно обновить.
  2. Проверьте, какие пакеты имеют критичные изменения.
  3. Обновите пакеты: mip update или mip update <package>.

В CI/CD:
Используйте mip outdated --json для автоматической проверки и отправки уведомлений в команду.

Связанные команды

  • mip update — обновление пакетов до последних версий.
  • mip install — установка пакетов.
  • mip audit — проверка безопасности пакетов.
  • mip list — просмотр установленных пакетов.
🧠 Как работает сравнение версий?

MIP использует семантическое версионирование для сравнения:

  1. Точное сравнение — версии сравниваются как строки (если current === latest, пакет актуален).

  2. Примеры:

    • 4.17.19 vs 4.17.21 → устарел.
    • 4.17.21 vs 4.17.21 → актуален.
    • ^4.17.0 vs 4.18.0 → устарел (если диапазон не включает новую версию).

  3. Важно: Команда сравнивает точные версии, а не диапазоны. Если в mip.yml указан диапазон, команда показывает версию, которая фактически установлена (из lock-файла или из глобального кэша).

Флаг --json:
Используйте для:

  • Автоматических уведомлений в чатах.
  • Интеграции с CI/CD пайплайнами.
  • Создания отчётов о состоянии зависимостей.
